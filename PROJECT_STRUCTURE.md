# 📂 비상기어 프로젝트 구조 상세 상세설명서

이 문서는 **비상기어(AiShift)** 프로젝트의 폴더 및 파일 구성과 각각의 역할을 상세히 설명합니다.

---

## 🏗️ 전체 구조 개요
이 프로젝트는 **Monorepo(모노레포)** 스타일로 구성되어 있으며, 프론트엔드(`client`)와 백엔드(`server`)가 독립된 폴더로 나누어져 있습니다.

---

## 📁 루트 폴더 (Root)
프로젝트의 최상위 경로로, 전체 설정 및 가이드 문서들이 위치합니다.

| 파일/폴더 | 역할 |
| :--- | :--- |
| **`.github/`** | GitHub 관련 설정. `workflows/deploy.yml` 파일은 자동 배포(Actions) 매뉴얼입니다. |
| **`.vscode/`** | VS Code 에디터 전용 설정. `tasks.json`을 통해 서버 자동 시작 등을 관리합니다. |
| **`client/`** | **리액트(프론트엔드) 프로젝트** 폴더입니다. (실제 화면 담당) |
| **`server/`** | **Node.js(백엔드) 프로젝트** 폴더입니다. (데이터 및 API 담당) |
| **`.gitignore`** | GitHub에 올리지 말아야 할 파일 목록(예: `node_modules`)을 정의합니다. |
| **`DEV_SERVER_GUIDE.md`** | 개발 서버 실행 방법에 대한 한글 가이드입니다. |
| **`HOSTING_GUIDE.md`** | 무료 호스팅(GitHub Pages 등) 배포 방법에 대한 가이드입니다. |
| **`NETWORK_ACCESS.md`** | 태블릿 등 다른 기기에서 접속하기 위한 네트워크 설정 가이드입니다. |
| **`start-dev.bat`** | 더블 클릭 한 번으로 개발 서버를 실행해 주는 윈도우 배치 파일입니다. |

---

## 📁 클라이언트 폴더 (client)
실제 사용자가 보는 웹사이트 소스 코드가 들어있는 곳입니다.

| 파일/폴더 | 역할 |
| :--- | :--- |
| **`dist/`** | **`npm run build` 실행 결과물.** 압축/최적화된 실제 배포용 파일들이 들어있습니다. |
| **`node_modules/`** | 개발에 필요한 수많은 외부 부품(라이브러리) 창고입니다. |
| **`public/`** | 빌드 과정 없이 그대로 배포될 정적 자산(파비콘 등)이 위치합니다. |
| **`src/`** | **핵심 소스 코드**가 들어있는 곳입니다. (아래 상세 설명 참고) |
| **`index.html`** | 웹사이트의 뼈대가 되는 메인 HTML 파일입니다. |
| **`package.json`** | 프로젝트 정보, 필요한 부품 목록, 실행 명령어(`scripts`)가 적힌 설정 파일입니다. |
| **`vite.config.js`** | 초고속 빌드 도구인 **Vite**의 설정 파일입니다. (배포 경로 등 설정) |

### 🔍 `client/src` 내부 구조
- **`assets/`**: 로고, 상품 이미지(SVG/PNG) 등 디자인 자산.
- **`components/`**: 버튼, 헤더, 푸터 등 재사용 가능한 UI 조각들.
- **`pages/`**: 메인 페이지, 로그인 페이지 등 각 화면 단위의 컴포넌트.
- **`App.jsx`**: 전체 페이지의 구조와 주소(Route)를 정의하는 메인 컴포넌트.
- **`main.jsx`**: 리액트 앱을 시작해서 HTML에 연결해 주는 입구 역할을 합니다.

## 📁 서버 폴더 (server)
데이터를 처리하고 관리하는 백엔드 영역입니다.

| 파일/폴더 | 역할 |
| :--- | :--- |
| **`package.json`** | 서버 전용 라이브러리 목록이 적혀 있습니다. |
| **`node_modules/`** | 서버 작동에 필요한 부품(Express 등) 창고입니다. |

## 💡 주요 파일 수정 가이드

1. **화면 디자인을 고치고 싶다면?**
   - `client/src/App.css` 또는 각 컴포넌트의 `.css` 파일을 수정하세요.
2. **새로운 상품을 추가하거나 페이지를 만들고 싶다면?**
   - `client/src/pages`에 새 파일을 만들고 `App.jsx`에 주소를 등록하세요.
3. **사이트를 새로 배포하고 싶다면?**
   - `client` 폴더 터미널에서 `npm run deploy`를 입력하세요. (자동화 설정 시 `git push`만으로 가능)
